## 主流程一：敏感查询与动态脱敏

1. 登录 `zhangming.dev`，密码 `123456`。
2. 进入 `SQL 安全` > `SQL审核工作台`。
3. 选择 `客户中心-MySQL生产库`。
4. 输入 SQL：

```sql
select name, id_card, mobile, address, account_balance from customer_info where city = '北京' limit 20;
```

5. 点击 `执行审核`。
6. 预期结果：命中敏感字段动态脱敏规则，审核放行，查询结果中的姓名、证件号、手机号、地址、余额被脱敏。
7. 进入 `SQL 安全` > `审计日志`，可以看到本次 SQL 执行和脱敏审计记录。

## 主流程二：DBA 明文授权对照

1. 切换到 `DBA 管理员 王强`，或重新登录 `wangqiang.dba`。
2. 进入 `SQL审核工作台`，仍选择 `客户中心-MySQL生产库`。
3. 输入与流程一相同的 SQL。
4. 将工作台右上角的脱敏开关切到 `明文`。
5. 点击 `执行审核`。
6. 预期结果：DBA 有明文授权，结果可以原样返回；审计日志仍记录敏感访问行为。

## 主流程三：DML 审批闭环

1. 登录 `lina.ops`，密码 `123456`。
2. 进入 `SQL审核工作台`。
3. 选择 `实时风控-GaussDB生产库`。
4. 输入 SQL：

```sql
update risk_event set event_status = 'CLOSED' where risk_level = 'LOW' and event_date < '2026-06-01';
```

5. 点击 `执行审核`。
6. 预期结果：生产库 DML 命中审批策略，系统弹出审批人选择框。
7. 选择 `DBA 管理员 王强`，生成审批单。
8. 切换到 `DBA 管理员 王强`。
9. 进入 `SQL 安全` > `DML审批` > `审批列表`。
10. 找到刚生成的审批单，点击 `通过`。
11. 切换回 `运维用户 李娜`。
12. 在审批列表中找到已通过的审批单，点击 `执行`。
13. 进入 `DML审批` > `执行列表`，确认生成执行记录。
14. 进入 `审计日志`，确认同一条 SQL 形成了 `提交审批`、`审批通过`、`DML执行` 三段审计链路。

## 主流程四：越权和阻断演示

1. 登录 `liuwei.risk`，密码 `123456`。
2. 进入 `SQL审核工作台`。
3. 尝试选择风控用户未授权的数据源，页面会自动限制到该用户可访问的数据源。
4. 在可访问数据源下输入高危 SQL：

```sql
select * from customer_info where 1=1;
```

5. 点击 `执行审核`。
6. 预期结果：命中恒真条件和注入特征，直接阻断，不生成查询结果。
7. 在 `审计日志` 中确认阻断记录、风险等级和 SQL 摘要。

## 主流程五：用户、角色和权限联动

1. 登录 `wangqiang.dba`。
2. 进入 `数据安全管理` > `用户管理`。
3. 查看用户列表，确认账号、部门、角色、数据源范围、字段权限一一对应。
4. 对任意非当前用户执行 `重置密码`，该用户密码会回到初始密码 `123456`。
5. 进入 `权限分配`，查看角色成员、业务模块、数据源范围和 SQL 操作范围。
6. 修改某个角色后，系统会同步刷新用户权限；切换到对应用户后，菜单和 SQL 工作台数据源范围会随之变化。

## 讲解口径

- 登录凭证、右上角用户切换、用户管理列表、权限角色和审批申请人都以同一份用户数据为准。
- SQL 审核会先校验用户数据源范围和 SQL 操作权限，再进行规则命中、风险定级、脱敏、审批或阻断。
- SELECT 放行后才展示查询结果；DML 必须先生成审批单，审批通过后由申请人执行。
- 审计日志记录 SQL 执行、阻断、提交审批、审批意见、DML 执行和脱敏状态。
- 所有数据为本地 mock 演示数据，页面操作会写入浏览器 `localStorage`；需要恢复种子数据时可以清理本地存储，或在审计日志页打开浏览器控制台执行 `window.__RKLINK_SQL_SECURITY_DEMO__?.resetAuditLogs()`。
